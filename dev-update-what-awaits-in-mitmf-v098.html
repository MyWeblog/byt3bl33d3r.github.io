Dev update: what awaits in MITMf v0.9.8

// under MITMf DevUpdate Python

I know, I know... it's been a while!

This is just to bring everyone (and with everyone I mean the 10 people who read this blog) up to speed on what's going to be heading your way in the next version bump. Currently, I'm in the process of porting MITMf's base proxy over to the awesome mitmproxy library, so goodbye Twisted!

This is kind of a big thing, it's why development on the old code base is at a stand still.. But it will make MITMf even more badass:

  • The code base will halved! (I'm not even kidding)
  • Plugins will use & benifit from mitmproxy's API
  • TLS/SSL interception support! (mind blowing, opens up a whole bunch of evi.. eer fun things to do)
  • Less dependencies!
  • HTTP/2 support!
  • Overall better code quality, which hopefully means more pull requests! Hint! ;)

Also MITMf was accepted into the Black Hat USA Arsenal this year, If you're in Vegas come check it out August 5!

To celebrate, I'm working on a couple of new plugins which I'll drop during the conference (If you've been following the project long enough you can probably guess what they are)

Finally, MITMf has been ported to a FruityWifi module! thanks to @xtr4nge for the PR and effort! I'm also expecting to see MITMf in the MANA toolkit and the Wifi Pineapple by the end of this year (fingers crossed!)
