MITMf V0.9.5: On the road to 1.0 and BHAsia 2015

// under MITMf Python

Wow a lot has happened in the past months!

I'm happy to announce that MITMf is now in the Kali Linux repositories (as some of you might already have noticed), also MITMf was accepted in the Black Hat Asia 2015 Arsenal Panel! Wohoo!

So now you can just run apt-get install mitmf in kali linux and your ready to rock and/or roll! Big thanks to @muts for all the help, patience and effort for setting up the framework for the distro!

So v0.9.5, under the hood, has had a major architecture redesign, lots of bug fixes and modularity improvements.This partially to get Responder up and running (more on that later) and to stop replicating code changes to the three versions of SSLstrip that MITMf used and to all of the plugins.

All three versions are now 'condensed' into a single version which you can find under the libs/sslstrip directory if you want to tinker with it.

There has been a dependency change regarding DNS tampering support, you can find all the details in the README.

MITMf now automatically writes everything sent to stdout into a log file called mitmf.log which conveniently you can find under the log/ directory.

If you run MITMf now you will see a couple of additional plugins:

  • Session hijacking plugin

If you just call it without additional arguments it will write the captured cookies to the log file. the --firefoxdb option will save cookies to a Firefox profile and the --mallory option will send the cookies to the "Mallory cookie injector browser extension" which you can download here.

I'm planning to add a lot more stuff to this plugin so consider this version an alpha.

  • Responder plugin

Let me just stop for a minute to explain how cool this is! Integrating Responder into MITMf opens up a bunch of future possibilities for the framework! Huge thanks to Lauren Gaffie (creator of Responder) for letting me do this.

So now you can man-in-the-middle WPAD, poison LLMNR, NBTNS and MDNS queries, and all the traffic will be redirected through the SSLstrip proxy so you can use the other plugins to manipulate the traffic!

  • Sniffer plugin

A port of Net-Creds by Dan McInerney

"Thoroughly sniffs passwords and hashes from the specified interface. Concatenates fragmented packets and does not rely on ports for service identification"

Awesome! So now while being in the middle, we can capture even more creds than before!

Additionally a LOT of bugs have been fixed... And I do mean a lot, especially when it comes to the SSLstrip+ and AppCachePoison plugins, so if they weren't working for you before I suggest you try them out again !

Specifically the SSLStrip+ code has been partially cleaned up and a bug that made certain websites go in an infinite redirect loop was fixed, the AppCachePoison plugin on the other hand was missing a very important function call which made it overlook some of the settings in the config file.

Overall I'm pretty happy with the shape that MITMf is taking, the next big thing will be actual SSL/TLS support, which will definitely take a while!

Also for all of the people wanting a tutorial , hopefully they will record the panel at BH Asia and put it on youtube in the coming months.
